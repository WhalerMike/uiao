## Methodology

1. Walk `src/uiao/canon/`, `src/uiao/modernization/`, and `src/uiao/schemas/`.
2. For each file, read frontmatter `publish_to_site` if present; otherwise apply ADR-068 path-pattern defaults.
3. For each file with `publish_to_site == true`, derive expected `.qmd` paths and check whether any of them appear in `docs/_quarto.yml`.
4. Report files with `publish_to_site == true` and no matching `.qmd` as gaps.

## How to close a gap

1. Decide the publication style (`narrative`, `include`, `reference`) and set it explicitly in the source document's frontmatter.
2. Author or generate the `.qmd` page at one of the expected paths (or any other path of your choice — then add it to `docs/_quarto.yml`).
3. Add the new `.qmd` to the sidebar in `docs/_quarto.yml`.
4. Re-run this scanner to confirm the gap is closed.

## How to suppress a false positive

If a document should NOT be published, add `publish_to_site: false` to its frontmatter with a brief justification in a comment. The scanner will then skip it on subsequent runs.
