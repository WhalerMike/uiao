# machine/adapters/

Machine contracts for adapter onboarding and integrations.

## Rules

- Only machine-parseable formats (`.json`, `.yaml`, `.yml`).
- No Markdown or prose.
- Adapter contracts must use semantic versioning (e.g., `adapter.entra.contract.v1.0.0.json`).
- Each adapter contract must have a corresponding human-facing explanation in `docs/architecture/`.
