Operational Handover: Modernization Atlas (V1.0)

Subject: Secure Orchestration & Automated Enforcement SOPs
Audience: CIO, CISO, CTO, PMO, Network Leadership, Identity Leadership, Executive Sponsors
Classification: CUI/FOUO
Version: 1.0

This document serves as the tactical guide for the Security Operations Center (SOC) to maintain, monitor, and govern the Modernization Atlas automated loop. It ensures that while the system is automated, it remains under human oversight and is resilient to the realities of a GCC-Moderate agency environment.

SOP 1: Monitoring Orchestrator Health

  • Primary Health Check (Sentinel): Monitor Atlas-Orchestrator-Heartbeat alert. Failure to receive success signal within 4 hours triggers P2 incident.

  • GitHub Actions Dashboard: Review WhalerMike/uiao-core Actions tab. Green = operational. Red = investigate Sync-Orchestrator run logs for API timeouts or credential failures.

  • Log Verification: Check MS Teams #Atlas-Operations channel for Sync Summary Adaptive Cards. Missing daily report indicates pipeline stall.

SOP 2: Handling False Positives from KQL Alerts

  1. Validate Identity Source: Cross-check HRIS status and Entra ID accountEnabled attribute.

  2. Verify Device Compliance: If device flagged Non-compliant, check Intune portal for specific reason.

  3. Tuning the KQL: If alert repeatedly fires on valid process, add user/device ID to watchlist_atlas_exceptions list in Sentinel.

SOP 3: Manual Override Procedures

  • ServiceNow Atlas_Ignore Tag: Add Atlas_Ignore: True to CI or User Record to prevent Atlas management.

  • Emergency Kill-Switch (GitHub): Tier 2 Engineer disables enforcement_orchestrator.yml workflow.

  • Manual Restoration: Use NetworkEnforcer script with –action remove flag or manually delete IP from Atlas_Quarantine_High_Risk DAG.

SOP 4: Escalation Paths

Tier Responsibility Trigger Condition
Tier 1 (SOC) | Initial Triage & Verification | Any Sentinel Atlas Alert or Teams Notification failure. |
Tier 2 (Eng) | Logic & API Troubleshooting | Pipeline failures, script errors, persistent false positives. |
Tier 3 (Lead) | Strategic Override / CISO Liaison | Broad network isolation events or suspected Atlas compromise. |

SOP 5: Runbook for Failure Scenarios

Scenario Immediate Action Resolution
Credential Expiry | Check GitHub Secrets for expired API keys. | Rotate keys in Vault; update GitHub Secrets. |
API Timeout | Verify network connectivity to .servicenowservices.gov. | Check TIC/VPN outages; retry GitHub Action. |
Rate Limiting | Check API usage logs. | Increase polling intervals. |
Orchestrator Crash | Check scripts for syntax errors. | Review most recent Git commit; rollback if necessary. |

Closing Protocol

SecOps must review the Atlas Sync Report during the daily morning briefing. Any Failed counts in the report require an immediate root-cause analysis (RCA) before the next scheduled sync.
